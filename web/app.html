Languages

Javascript & NodeJS

I've used Javascript in almost any scope you can imagine. I'm very fluent with jQuery and DOM manipulation, although more recently I've been building entire Javascript driven single-page web applications using technologies such as AngularJS and NodeJS. Javascript has been used in essentially every web application I've worked and is one of my favourite languages to code in.

HTML and CSS

Being a web application developer, it's hard to not obtain an extensive knowledge of HTML and my case is no exception. I've also had the pleasure of developing modern web applications that utilize new technolgoies; such as HTML5 and CSS3, while remaining functional in older browsers (as far back as IE6). Most recently I've been utilizing CSS pre-compilers such as Sass and it's hard to imagine going back to anything else.

PHP

One of my strongest languages. I've been coding in PHP since the first day I could code. Object-Oriented and frameworks such as Zend, Silex and CakePHP are second-nature to me. From single-page sites to large-scale web applications, I've done them all.

Python

By far my favourite language to code in, largely attributed to it's beautiful syntax. When building a web application in Python, I've found Django the most satisfying.

Git

Always my first choice for version control, from personal to large-scale business projects. My knowledge of Git is extensive and I'm a strong advocate of the pull request, peer review style of development.

Swift & Objective-C

Although I don't know these technologies well yet, I never stop learning and my goal to teach myself Swift & Objective-C so I can code my first native iOS application.

MySql, Postgres, MongoDB and Redis

Every project is different, but usually I choose one or two of these databases depending on the scale or requirements of the application. Generally I'm a big advocate of keeping the database strictly as a datastore and using your application for all the complicated business logic. I've known many people who are persistent on putting far too much application logic straight into the database layer through features like stored procedures, only to see it explode and quickly become unmaintable. In terms of relational vs NoSql, it really is a decision you make on a case-by-case basis.

Unit and Functional Testing

It's one of those things you don't realize how important it is until you actually start doing it. And once you start, you won't go back. Every application I develop now is always done with testing in mind. Test first, code second. Being a believer of continuous deployment makes it easy to see the value in unit tests. You won't find me coding without them.

Synapsie

Built in 2011

I built Synapsie as an opportunity to learn Python web development and it accomplished this nicely. It was a great learning experiment that introduced me to the increasingly popular Django Framework as well as Facebook Connect.

For the development of Synapsie, I used a Vagrant Virtual Machine along with Puppet for server configuration. I implemented a simple RESTful API and the majority of the UX is accomplished using jQuery and calls to the API. Although I'm not much of a designer, I did design Synapsie myself using Adobe Illustrator and I think it turned out well.

CoursePark

From 2008 — Present

I have worked on CoursePark ever since I was hired at Bluedrop. CoursePark was little more than a thought scribbled on paper when we started development and has since become a cornerstone product from Bluedrop with 10s of thousands of active users.

About a year into development I was promoted to Lead Programmer of CoursePark and my team and I have never stopped improving the application. We continue to implement the latest technologies such as the newly established eLearning API called TinCan and we are in the process of expanding to a mobile offering. As Lead on the project, I pushed us towards a Continuous Deployment model utilizing Unit & Functional Tests along with Pull Request style peer review. We also migrated from using SVN to GIT for version control which showed substantial performance improvements and allowed us to obtain the rapid development and release workflow we needed.

We built CoursePark on PHP, Postgres and the Zend Framework using the MVC architecture pattern. Our servers have been architected to provide a reliable uptime, performance and avoid any single point of failure. Vagrant VMs are used for local development along with Puppet scripts for managing development and production configurations.
